# 智能垃圾分类小程序

> 原项目地址：https://github.com/qi19901212/Garbage

## 完成的基础题

### 基础题1：扩充垃圾数据库
新增 50 条校园特色垃圾条目，涵盖四种分类：
- **可回收物（14条）**：废旧教科书、废草稿纸、废旧校园卡、废旧鼠标/键盘等
- **湿垃圾（13条）**：食堂剩饭剩菜、过期酸奶、吃剩的关东煮等
- **有害垃圾（10条）**：实验室废液、废弃化学试剂瓶、废弃水银体温计等
- **干垃圾（13条）**：用过的餐巾纸、一次性餐盒/筷子、烟蒂等

同时编写了 `batchImport` 云函数用于将新数据批量导入云数据库。

### 基础题2：新增"分类指南"页面
新增独立的分类指南页面（`pages/guide/guide`），通过底部 tabBar "指南"入口进入，内容包括：
- **Tab 切换**：可回收物 / 有害垃圾 / 湿垃圾 / 干垃圾四个分类标签
- **判断技巧**：每类垃圾的投放要点和判断方法
- **常见误区**：每类垃圾 4 个典型易混淆案例（如"大骨头不是湿垃圾"）

## 运行方法

1. 下载并安装 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
2. 克隆本仓库：`git clone <仓库地址>`
3. 在微信开发者工具中导入项目（选择项目根目录）
4. AppID 可使用测试号（工具会自动分配）
5. 如需云开发功能，在开发者工具中开通云开发环境，并部署 `cloudfunctions` 目录下的云函数
6. 运行 `batchImport` 云函数导入新增的 50 条垃圾数据

## 自我总结

本次选择了垃圾分类项目，完成了两道基础题。扩充数据库时重点选取了校园场景下的常见废弃物，覆盖实验室、食堂、教室、宿舍等场景。分类指南页面采用 Tab 切换设计，将四类垃圾的判断技巧和常见误区整合在一个页面中，方便用户快速查阅。开发中遇到的主要问题是 tabBar 配置需要保持 iconPath 一致性，以及 WXML 中 wx:for 嵌套需要用 wx:for-item 避免变量名冲突。
